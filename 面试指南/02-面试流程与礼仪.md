# 面试流程与礼仪

## 一、互联网公司常见面试流程

### 1.1 典型面试流程总览

```
简历筛选 → HR电话初筛 → 技术一面 → 技术二面 → 交叉面/架构面 → HR终面 → Offer沟通
```

不同公司流程有差异，但大体遵循以上节奏。有些公司会加笔试或机试环节。

### 1.2 各轮面试详解

#### HR电话初筛（15-20分钟）

**目的**：确认基本信息匹配度，筛掉明显不合适的候选人。

**常见问题**：
- 目前在职还是离职？
- 期望薪资和到岗时间？
- 为什么考虑换工作？
- 简单介绍一下你的工作经历

**应对策略**：
- 语气积极、态度诚恳
- 薪资可以给一个范围，不要把自己锁死
- 离职原因要正面表达（寻求更大的发展空间、技术挑战等），不要抱怨前公司
- 到岗时间如实说，一般1个月内比较理想

#### 技术一面（45-60分钟）

**目的**：考察基础技术能力和项目经验。

**面试官**：通常是未来的直属leader或同级资深工程师。

**内容构成**：
| 模块 | 占比 | 内容 |
|------|------|------|
| 自我介绍 | 5% | 1-2分钟 |
| 项目深挖 | 40% | 挑1-2个项目追问技术细节 |
| 基础知识 | 40% | Java基础、集合、并发、JVM、数据库、缓存等 |
| 算法题 | 10% | 1道中等难度算法题 |
| 反问环节 | 5% | 你有什么想问的 |

**应对策略**：
- 项目要讲清楚你做了什么，不是团队做了什么
- 基础知识要扎实，不能含糊
- 算法题注重思路表达，边写边说

#### 技术二面（45-60分钟）

**目的**：考察技术深度、架构能力和解决问题的思维。

**面试官**：通常是更高级别的技术leader或架构师。

**内容构成**：
| 模块 | 占比 | 内容 |
|------|------|------|
| 项目架构 | 50% | 架构决策、技术选型理由、踩过的坑 |
| 系统设计 | 25% | 场景题，如设计一个消息推送系统 |
| 技术深度 | 15% | 某个技术点的深入追问（如Redis底层、MySQL索引原理） |
| 开放性问题 | 10% | 技术视野、学习方法、职业规划 |

**应对策略**：
- 重点准备"为什么这样设计"而不只是"怎么实现的"
- 系统设计题要有结构化的思考框架
- 展示你的技术判断力，而不只是执行力

#### 交叉面/加面（30-45分钟）

**目的**：其他团队的技术leader评估你的综合能力，避免单一面试官的偏差。

**特点**：
- 面试官对你的项目不熟悉，更关注通用能力
- 可能会出更开放的问题
- 考察沟通能力和逻辑思维

#### HR终面（20-30分钟）

**目的**：评估文化匹配度、稳定性，确认薪资期望。

**常见问题**：
- 你最大的优点和缺点是什么？
- 你和同事/leader有过分歧吗？怎么处理的？
- 你的职业规划是什么？
- 你还在看其他机会吗？
- 期望薪资是多少？

**应对策略**：
- 缺点要说真实的，但要附带改进措施
- 分歧问题要展示你的沟通能力和大局观
- 职业规划要和岗位方向一致
- 有其他offer可以适当透露，但不要用来威胁

## 二、面试前的准备

### 2.1 信息收集

面试前至少做好以下功课：

| 维度 | 具体内容 | 渠道 |
|------|----------|------|
| 公司背景 | 主营业务、发展阶段、技术栈 | 官网、脉脉、天眼查 |
| 团队情况 | 团队规模、技术方向、在做什么产品 | JD描述、脉脉、面试官LinkedIn |
| 岗位要求 | 必须技能、加分项、职级范围 | JD原文，逐条对照 |
| 面试风格 | 面试轮数、是否有笔试、常见题型 | 牛客网面经、脉脉 |
| 薪资范围 | 该公司该职级的薪资区间 | 脉脉、offershow、职级对标 |

### 2.2 技术准备优先级

根据面试频率和考察权重，建议按以下优先级准备：

```
P0（必须掌握）：Java基础、集合、并发、JVM、MySQL、Redis、Spring
P1（高频考察）：设计模式、计算机网络、操作系统、消息队列
P2（加分项）  ：系统设计、分布式理论、ES、Netty
P3（按岗位）  ：特定技术栈（如你的IoT、AI方向）
```

### 2.3 物料准备

- [ ] 简历PDF版（文件名：姓名-岗位.pdf）
- [ ] 自我介绍练习到自然流畅
- [ ] 项目经历的STAR描述准备好
- [ ] 每个项目准备3个可能被追问的技术点
- [ ] 准备2-3个反问问题
- [ ] 电脑/手机充好电，网络稳定（远程面试）
- [ ] 安静的面试环境

### 2.4 面试前30分钟检查表

临近面试时不要再大范围学习新内容，只做"唤醒记忆"。

| 时间 | 做什么 | 目标 |
|------|--------|------|
| T-30分钟 | 打开简历和JD，对照关键词 | 确认今天要强调什么 |
| T-25分钟 | 复述1遍自我介绍 | 控制在1-2分钟 |
| T-20分钟 | 快速过4个核心项目的一句话版本 | 防止项目讲散 |
| T-15分钟 | 准备纸笔/会议软件/共享屏幕 | 避免临场慌乱 |
| T-10分钟 | 看反问问题 | 结尾有话可问 |
| T-5分钟 | 深呼吸，关闭无关通知 | 进入沟通状态 |

### 2.5 根据岗位调整当天重点

面试不是把所有经历平均讲一遍，而是根据岗位选择主线。

以视频云平台/IoT平台开发岗位为例，当天重点建议：

| 面试内容 | 优先强调 |
|----------|----------|
| 自我介绍 | IoT平台、设备接入、推送系统、高并发消息 |
| 项目深挖 | IoT SaaS平台、云平台推送系统 |
| 技术基础 | Java并发、Netty/NIO、MySQL、Redis、消息队列 |
| 系统设计 | 设备接入系统、消息推送系统、OTA/设备影子 |
| 行为问题 | 线上问题定位、跨团队协作、稳定性建设 |

## 三、面试中的沟通技巧

### 3.1 回答问题的基本框架

#### 知识类问题（如：HashMap的底层原理）

```
结论先行 → 展开细节 → 补充延伸
```

示例：
> "HashMap底层是数组+链表+红黑树的结构。（结论）
> 默认初始容量16，负载因子0.75，当链表长度超过8且数组长度>=64时转为红黑树。（细节）
> JDK 1.7用的是头插法，1.8改成了尾插法，主要是为了解决并发扩容时的死循环问题。（延伸）"

#### 项目类问题（如：你的推送系统是怎么设计的）

```
背景 → 方案 → 难点 → 结果
```

#### 场景类问题（如：如何设计一个秒杀系统）

```
需求确认 → 整体方案 → 核心设计 → 优化与取舍
```

### 3.2 不会的问题怎么办

**最差的回答**："这个我不会。"（句号结束）

**好的回答**：
1. 说出你知道的相关内容
2. 说出你的推测和思考方向
3. 坦诚表示这块了解不深，但表达学习意愿

示例：
> "这个问题我没有深入研究过，但根据我对XX的理解，我猜测可能是YY的机制。回去之后我会深入学习一下。"

### 3.3 高频沟通场景应对

| 场景 | 建议 |
|------|------|
| 面试官打断你 | 不要慌，可能是时间有限，简洁回答核心点 |
| 面试官追问"还有呢" | 说明他对你的回答感兴趣，继续深入 |
| 面试官说"我们换个话题" | 可能你答偏了或答得太浅，不要纠结，调整状态 |
| 面试官沉默 | 可能在记录，等几秒，不要急着填充 |
| 被问到薪资 | 给范围而非精确数字，表示可以谈 |
| 被问"你有什么想问的" | 一定要问，准备2-3个有质量的问题 |

### 3.4 面试中的控场技巧

#### 主动确认问题

遇到开放问题，不要急着答：

> "我理解您主要想了解的是XX，对吗？如果是这个方向，我可以从背景、方案和实际效果三部分说。"

#### 回答太长时主动收束

如果发现自己讲了很多，可以这样收：

> "我先讲到这里。这个问题如果继续深入，可以展开到XX和YY两个细节，您看更想了解哪一块？"

#### 面试官追问很深时

不要慌，说明他对这个点感兴趣：

> "这个点我拆开说：第一是设计目标，第二是具体实现，第三是上线后遇到的问题。"

#### 回答后补一句总结

复杂问题最后用一句话落地：

> "所以这个方案的核心是用异步消息削峰，用幂等和补偿保证最终一致，用监控把链路可观测起来。"

## 四、面试礼仪

### 4.1 现场面试礼仪

| 环节 | 注意事项 |
|------|----------|
| 着装 | 互联网公司不需要西装，干净整洁的休闲装即可 |
| 时间 | 提前10-15分钟到达，不要迟到也不要太早 |
| 进门 | 敲门、微笑、主动打招呼 |
| 坐姿 | 坐直，不要翘二郎腿，不要抖腿 |
| 眼神 | 看面试官，不要一直低头或看别处 |
| 手机 | 静音或关机 |
| 结束 | 感谢面试官的时间，礼貌告别 |

### 4.2 远程面试礼仪

| 环节 | 注意事项 |
|------|----------|
| 环境 | 安静、光线充足、背景整洁 |
| 设备 | 提前测试摄像头、麦克风、网络 |
| 软件 | 提前安装好面试工具（腾讯会议、飞书、Zoom等） |
| 网络 | 有线网络优先，准备手机热点备用 |
| 共享屏幕 | 关闭无关窗口和通知，桌面保持整洁 |
| 写代码 | 提前熟悉在线编程平台（牛客、CodePair等） |
| 延迟 | 说完后停顿1-2秒再继续，避免抢话 |

#### 远程面试突发情况话术

| 场景 | 话术 |
|------|------|
| 网络卡顿 | "不好意思，我这边网络有点卡，刚才最后一句没有听清，能麻烦您再说一遍吗？" |
| 听不清问题 | "这个问题我怕理解偏了，我复述一下，您看是不是这个意思。" |
| 共享屏幕失败 | "我先口头描述思路，同时尝试重新共享。如果还不行，我可以用文字把结构发出来。" |
| 编程平台异常 | "平台好像暂时无法运行，我先把思路和代码写完整，再手动走一遍测试用例。" |

### 4.3 面试后的跟进

- **当天**：可以给HR或猎头发一条感谢消息，表达面试感受和意愿
- **3天内**：如果没有收到反馈，可以主动询问进度
- **1周后**：如果仍无反馈，大概率没过，调整心态继续下一场
- **收到offer**：不要立刻答复，争取1-3天考虑时间

## 五、反问环节的高质量问题

反问环节是你了解团队和展示思考深度的机会，不要说"没什么想问的"。

### 推荐问题（按面试轮次）

**技术面可以问**：
- 团队目前在做什么方向？技术栈是怎样的？
- 这个岗位日常的工作内容是什么？
- 团队目前面临的最大技术挑战是什么？
- 新人入职后大概多久能独立负责模块？
- 设备接入、消息通道、流媒体服务这几块，目前团队最希望新人先参与哪一块？
- 长连接稳定性和设备规模方面，团队现在最关注的指标是什么？
- 团队在MQTT/WebSocket/WebRTC这些协议上，是自研接入层还是基于现有Broker做二次开发？

**HR面可以问**：
- 这个岗位的晋升路径是怎样的？
- 团队的工作节奏是怎样的？（加班情况的委婉问法）
- 公司的培训和学习机制是怎样的？

### 不建议问的问题

- 加班多不多？（太直接，换个问法）
- 你觉得我表现怎么样？（让面试官尴尬）
- 这个岗位为什么招人？（可能涉及敏感信息）
- 薪资能给多少？（技术面不要问，留给HR面）

## 六、薪资谈判策略

### 6.1 谈薪的时机

- 技术面不要主动提薪资
- HR初筛时给一个范围即可
- 正式谈薪在HR终面或offer阶段

### 6.2 谈薪的原则

| 原则 | 说明 |
|------|------|
| 知己知彼 | 提前了解该公司该职级的薪资范围 |
| 先让对方出价 | "想先了解一下贵司的薪资结构" |
| 给范围不给精确值 | "我的期望在XX-XX之间" |
| 有理有据 | 基于当前薪资、市场行情、个人能力来谈 |
| 不要撒谎 | 虚报当前薪资可能被背调发现 |
| 综合考虑 | 不只看月薪，还要看年终、股票、福利、成长空间 |

### 6.3 常见谈薪话术

**被问当前薪资**：
> "我目前的总包大概在XX左右，但我更看重新机会的发展空间和技术挑战。"

**被问期望薪资**：
> "结合我的经验和市场行情，我的期望在XX-XX之间，具体可以根据整体package来谈。"

**对方出价偏低**：
> "感谢offer，但这个数字和我的预期有一些差距。考虑到我在XX方面的经验，能否再沟通一下？"

**有多个offer**：
> "目前手上有其他机会在对比，贵司是我比较倾向的选择，希望在薪资上能更有竞争力。"
