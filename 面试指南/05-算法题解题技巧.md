# 算法题解题技巧

## 一、面试算法题的定位

### 1.1 面试算法 ≠ 竞赛算法

面试中的算法题和ACM竞赛完全不同：

| 维度 | 面试算法 | 竞赛算法 |
|------|----------|----------|
| 难度 | Easy-Medium为主 | Medium-Hard |
| 考察重点 | 思路表达 + 代码质量 | 正确性 + 效率 |
| 时间 | 15-25分钟/题 | 2-5小时/多题 |
| 沟通 | 需要边做边说 | 独立完成 |
| 代码风格 | 命名规范、可读性 | 能跑就行 |

核心认知：**面试算法考的是你的思维过程和编码能力，不是你会不会解偏题怪题。**

### 1.2 Java后端面试的算法考察范围

根据面试频率排序：

| 优先级 | 类别 | 高频题型 |
|--------|------|----------|
| P0 | 数组/字符串 | 双指针、滑动窗口、前缀和 |
| P0 | 链表 | 反转、合并、环检测、快慢指针 |
| P0 | 二叉树 | 遍历、层序、LCA、路径问题 |
| P0 | 哈希表 | 两数之和、字母异位词、LRU缓存 |
| P1 | 二分查找 | 标准二分、旋转数组、搜索边界 |
| P1 | 动态规划 | 背包、最长子序列、路径问题 |
| P1 | 栈/队列 | 有效括号、单调栈、最小栈 |
| P2 | BFS/DFS | 岛屿问题、图遍历、拓扑排序 |
| P2 | 排序 | 快排、归并、TopK问题 |
| P2 | 贪心 | 区间调度、跳跃游戏 |
| P3 | 回溯 | 全排列、组合、N皇后 |
| P3 | 位运算 | 只出现一次的数字、位计数 |

> 建议：P0类别的题目要做到闭眼能写，P1要熟练，P2了解套路，P3看时间。

## 二、解题框架：五步法

### 2.1 总览

拿到一道算法题，按以下五步走：

```
Step 1：理解题意（1-2分钟）
Step 2：举例验证（1分钟）
Step 3：说出思路（2-3分钟）
Step 4：编写代码（8-12分钟）
Step 5：测试验证（2-3分钟）
```

### 2.2 Step 1：理解题意

**不要急着写代码**。先确认：

- 输入是什么？输出是什么？
- 有什么约束条件？（数组是否有序？元素是否重复？）
- 边界情况？（空数组？只有一个元素？）

如果题意不清楚，**主动问面试官**：

> "请问这里的XX是指YY吗？"
> "输入数据的范围大概是多少？"

### 2.3 Step 2：举例验证

用1-2个简单例子手动走一遍，确认你理解了题意：

```
输入：[2, 7, 11, 15], target = 9
预期输出：[0, 1]  （因为 2 + 7 = 9）
```

这一步还能帮你发现思路。

### 2.4 Step 3：说出思路

**这是最关键的一步**。在写代码之前，先把思路说给面试官听：

模板：
> "这道题我的思路是用[方法名]。核心想法是[一句话概括]。时间复杂度是O(X)，空间复杂度是O(X)。"

示例（两数之和）：
> "这道题我用哈希表来做。遍历数组，对每个元素检查 target - nums[i] 是否已经在哈希表中。如果在，直接返回；如果不在，把当前元素存入哈希表。时间复杂度O(n)，空间复杂度O(n)。"

面试官可能会：
- 说"可以，写吧" → 开始写代码
- 说"有没有更优的方案" → 继续优化思路
- 给提示 → 顺着提示调整

### 2.5 Step 4：编写代码

写代码时的注意事项：

| 要点 | 说明 |
|------|------|
| 边写边说 | "这里我先处理边界情况"、"这个循环是在做XX" |
| 变量命名 | 用有意义的名字：`left/right`、`slow/fast`、`count` |
| 先写框架 | 先写出函数签名和整体结构，再填充细节 |
| 边界先行 | 先处理空值、空数组等边界情况 |
| 不要纠结 | 小的语法问题不要纠结太久，面试官不会因为少个分号扣分 |

### 2.6 Step 5：测试验证

写完代码后，**不要说"写完了"就停下**。主动做：

1. 用题目给的示例走一遍代码
2. 想一个边界用例测试（空输入、单元素、全相同等）
3. 说出时间和空间复杂度

> "我用示例数据走一遍：[2,7,11,15], target=9。i=0, num=2, 哈希表中没有7，存入{2:0}。i=1, num=7, 哈希表中有2（9-7=2），返回[0,1]。结果正确。"

## 三、常见算法套路速查

### 3.1 双指针

**适用场景**：有序数组、链表操作、原地修改

**核心思想**：用两个指针从不同位置或不同速度遍历，减少嵌套循环。

| 类型 | 说明 | 典型题目 |
|------|------|----------|
| 对撞指针 | 左右两端向中间靠拢 | 两数之和II、盛最多水的容器、三数之和 |
| 快慢指针 | 一快一慢检测环或找中点 | 链表环检测、链表中点、快乐数 |
| 同向指针 | 同方向移动，一个快一个慢 | 移除元素、删除有序数组重复项 |

**解题模板**：
```java
// 对撞指针
int left = 0, right = arr.length - 1;
while (left < right) {
    if (满足条件) return result;
    else if (需要右移左指针) left++;
    else right--;
}
```

### 3.2 滑动窗口

**适用场景**：连续子数组/子串问题

**核心思想**：维护一个窗口，右边界扩张探索，左边界收缩优化。

**识别信号**：题目中出现"连续"、"子数组"、"子串"、"最长/最短"。

| 类型 | 说明 | 典型题目 |
|------|------|----------|
| 固定窗口 | 窗口大小固定 | 定长子数组最大和 |
| 可变窗口 | 窗口大小动态调整 | 无重复字符的最长子串、最小覆盖子串 |

**解题模板**：
```java
int left = 0;
Map<Character, Integer> window = new HashMap<>();
for (int right = 0; right < s.length(); right++) {
    char c = s.charAt(right);
    window.merge(c, 1, Integer::sum);
    
    while (窗口需要收缩) {
        char d = s.charAt(left);
        window.merge(d, -1, Integer::sum);
        left++;
    }
    // 更新结果
}
```

### 3.3 二分查找

**适用场景**：有序数组查找、搜索边界、最小化最大值

**核心思想**：每次排除一半的搜索空间。

**识别信号**：有序、查找、O(log n)要求。

| 变体 | 说明 | 关键区别 |
|------|------|----------|
| 标准二分 | 查找精确值 | `left <= right` |
| 左边界 | 查找第一个>=target的位置 | `right = mid` |
| 右边界 | 查找最后一个<=target的位置 | `left = mid` |

**易错点**：
- `mid = left + (right - left) / 2` 防止溢出
- 循环条件是 `<=` 还是 `<`，取决于搜索区间是闭区间还是左闭右开
- 更新时是 `mid + 1` 还是 `mid`，想清楚排除的是哪一半

### 3.4 BFS/DFS

**适用场景**：树/图遍历、连通性、最短路径、层序遍历

| 算法 | 数据结构 | 适用场景 |
|------|----------|----------|
| BFS | 队列 | 最短路径、层序遍历 |
| DFS | 栈/递归 | 路径搜索、连通分量、回溯 |

**BFS关键点**：
- 用队列实现
- 需要visited集合防止重复访问
- 按层遍历时记录每层的size

**DFS关键点**：
- 递归或显式栈实现
- 注意递归终止条件
- 回溯时恢复状态

### 3.5 动态规划

**适用场景**：最优解问题、计数问题、可行性问题

**核心思想**：将大问题拆解为子问题，通过子问题的解推导出大问题的解。

**解题四步法**：
```
1. 定义状态：dp[i] 代表什么含义
2. 状态转移：dp[i] 和 dp[i-1]（或其他状态）的关系
3. 初始条件：dp[0] 等于什么
4. 遍历顺序：从前往后还是从后往前
```

**常见DP类型**：

| 类型 | 典型题目 | 状态定义 |
|------|----------|----------|
| 线性DP | 最长递增子序列、最大子数组和 | dp[i]：以i结尾的最优解 |
| 背包DP | 0-1背包、完全背包、零钱兑换 | dp[i][j]：前i个物品容量j的最优解 |
| 区间DP | 最长回文子串、戳气球 | dp[i][j]：区间[i,j]的最优解 |
| 路径DP | 不同路径、最小路径和 | dp[i][j]：到达(i,j)的最优解 |
| 字符串DP | 编辑距离、最长公共子序列 | dp[i][j]：s1前i个和s2前j个的最优解 |

**识别信号**：
- "最多"、"最少"、"多少种方式" → 大概率是DP
- 能拆成子问题，且子问题有重叠 → 适合DP
- 如果暴力解是指数级复杂度 → 考虑DP优化

### 3.6 回溯

**适用场景**：排列、组合、子集、棋盘问题

**核心思想**：穷举所有可能，遇到不满足条件的分支就剪枝回退。

**解题模板**：
```java
void backtrack(路径, 选择列表) {
    if (满足结束条件) {
        result.add(路径);
        return;
    }
    for (选择 : 选择列表) {
        做选择;       // 将选择加入路径
        backtrack(路径, 选择列表);
        撤销选择;     // 将选择从路径移除
    }
}
```

**剪枝技巧**：
- 排序后跳过重复元素
- 用visited数组标记已使用的元素
- 提前判断当前路径是否可能达到目标

### 3.7 单调栈

**适用场景**：下一个更大/更小元素、柱状图最大矩形

**核心思想**：维护一个单调递增或递减的栈，遇到破坏单调性的元素时处理栈顶。

**典型题目**：每日温度、下一个更大元素、柱状图中最大的矩形、接雨水

## 四、面试中的时间管理

### 4.1 不同难度的时间分配

| 难度 | 预期时间 | 时间分配 |
|------|----------|----------|
| Easy | 10-15分钟 | 理解1min + 思路2min + 编码5-8min + 测试2min |
| Medium | 15-25分钟 | 理解2min + 思路3-5min + 编码8-12min + 测试3min |
| Hard | 25-35分钟 | 理解2min + 思路5-8min + 编码12-18min + 测试3min |

### 4.2 时间紧迫时的策略

**5分钟还没思路**：
- 先说暴力解法，再尝试优化
- 向面试官要提示，不丢人

**代码写到一半发现思路有问题**：
- 不要硬写下去
- 停下来说"我发现这个思路有问题，XX情况处理不了，我换个思路"
- 面试官会欣赏你的自我纠错能力

**时间快到了代码没写完**：
- 把剩余部分用伪代码或注释说明
- 说清楚完整的思路和复杂度
- "如果时间充裕，这里我会用XX方式实现"

## 五、卡住了怎么办

### 5.1 系统化的破局方法

当你完全没有思路时，按以下顺序尝试：

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1 | 重读题目 | 是否遗漏了关键条件？ |
| 2 | 举极端例子 | 最小输入、最大输入、全相同、全不同 |
| 3 | 暴力枚举 | 先想最笨的方法，再优化 |
| 4 | 分类讨论 | 这道题属于哪个类型？数组？树？图？ |
| 5 | 逆向思考 | 从结果倒推，需要什么条件？ |
| 6 | 简化问题 | 先解决简化版本，再推广 |
| 7 | 求助面试官 | "能给我一个提示吗？" |

### 5.2 从暴力到优化的思路

大部分面试题的优化路径：

```
暴力枚举 O(n²) 或 O(2^n)
  ↓ 用哈希表空间换时间
优化到 O(n)
  ↓ 或者用排序+双指针
优化到 O(n log n)
  ↓ 或者用二分查找
优化到 O(log n)
```

**示例：两数之和**
- 暴力：两层循环，O(n²)
- 优化：哈希表，一次遍历，O(n)

**示例：在有序数组中查找目标**
- 暴力：线性扫描，O(n)
- 优化：二分查找，O(log n)

### 5.3 向面试官求助的正确方式

求助不丢人，但要展示你已经思考过：

❌ "这题我不会，能给个提示吗？"

✅ "我目前想到的是用XX方法，但在处理YY的情况时遇到了困难。能给我一个方向上的提示吗？"

面试官给提示后：
- 认真听，不要急着打断
- 复述一遍确认理解："您的意思是用XX来处理YY，对吗？"
- 顺着提示继续推进

## 七、复杂度分析速查

面试中每道题都要说出复杂度，以下是常见复杂度的直觉判断：

| 复杂度 | 直觉判断 | 典型算法 |
|--------|----------|----------|
| O(1) | 常数操作 | 哈希表查找、数组索引访问 |
| O(log n) | 每次排除一半 | 二分查找 |
| O(n) | 遍历一次 | 线性扫描、哈希表解法 |
| O(n log n) | 排序 | 归并排序、快速排序 |
| O(n²) | 两层嵌套循环 | 暴力枚举、冒泡排序 |
| O(2^n) | 所有子集 | 回溯、暴力搜索 |
| O(n!) | 所有排列 | 全排列 |

**面试中的表达**：
> "这个解法的时间复杂度是O(n)，因为只遍历了一次数组。空间复杂度是O(n)，因为用了一个哈希表来存储已遍历的元素。"
