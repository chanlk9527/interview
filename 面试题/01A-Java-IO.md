# 01A - Java IO

---

## 一、IO 基础与资源管理

1. 字节流、字符流、缓冲流分别解决什么问题？在文件、网络、日志、接口响应中如何选择？
2. 字符编码为什么经常导致乱码？服务间传输、文件导入、数据库读写中如何统一编码策略？
3. `try-with-resources`、`AutoCloseable`、`flush()`、`close()` 分别在资源释放中承担什么责任？
4. Java IO 中的装饰器模式带来了什么灵活性？过度包装时又会带来哪些可读性问题？
5. 大文件读取为什么不能简单 `readAllBytes()`？如何根据文件大小、处理方式和内存限制选择方案？
6. 临时文件、文件权限、路径拼接在安全上有哪些风险？如何防止路径穿越和敏感文件泄露？

## 二、文件 API 与大文件处理

1. `Path`、`Files`、`File` 的定位是什么？新代码中为什么更推荐 `java.nio.file` API？
2. 大文件逐行处理、分块上传、断点续传、批量导入分别适合哪些读写模式？
3. `MappedByteBuffer` 适合什么场景？它在内存释放、文件锁定、地址空间上有哪些坑？
4. 文件锁 `FileLock` 能解决哪些跨进程并发问题？它有哪些平台相关限制？
5. `WatchService` 文件监听适合哪些场景？为什么不能把它当成强一致事件系统？
6. 线上文件处理任务如何做限速、重试、幂等和失败恢复？

## 三、网络 IO 模型

1. BIO、NIO、AIO 的核心差异是什么？实际项目中为什么更多使用 Netty、Servlet 容器或成熟客户端库？
2. Socket 读写中的粘包、半包、超时、连接关闭分别如何处理？
3. Selector、Channel、Buffer 的协作模型是什么？为什么直接写 NIO 代码容易出错？
4. 堆内 Buffer 和直接内存 Buffer 如何取舍？直接内存泄漏或占用过高怎么排查？
5. Reactor 模式如何支撑高并发连接？单 Reactor、多 Reactor、主从 Reactor 的区别是什么？
6. Linux `select`、`poll`、`epoll` 的差异会如何影响 Java 网络框架的设计？

## 四、零拷贝与数据传输

1. 传统文件传输经过哪些用户态和内核态拷贝？零拷贝减少了什么成本？
2. `FileChannel.transferTo/transferFrom`、`sendfile`、`mmap` 分别适合哪些传输场景？
3. Kafka、Netty、静态文件下载为什么会关注零拷贝？它对业务接口一定有收益吗？
4. 零拷贝和 TLS 加密、压缩、限速、审计日志之间有什么冲突或限制？
5. 做大文件下载接口时，如何设计 Range、缓存头、限流、断连处理和观测指标？

## 五、序列化与数据格式

1. Java 原生序列化为什么在现代服务中应谨慎使用？性能、兼容性和安全风险分别是什么？
2. JSON、Protobuf、Avro、Kryo、Hessian 在可读性、跨语言、Schema 演进、性能上如何取舍？
3. 反序列化漏洞的本质是什么？如何通过白名单、禁用危险类型、依赖治理降低风险？
4. 接口字段新增、删除、重命名时，如何保证序列化格式的前后兼容？
5. RPC、MQ、缓存、数据库 JSON 字段分别对序列化方案有什么不同要求？

## 六、流式处理与背压

1. 为什么高吞吐文件或网络处理需要考虑背压？生产者过快会导致哪些资源问题？
2. `InputStream`/`OutputStream`、Iterator、Stream API、Reactive Streams 在流式处理上的差异是什么？
3. 上传文件边读边处理时，如何避免把完整文件放进内存？
4. 下载或转发大响应时，如何处理客户端慢连接、下游超时和取消？
5. WebFlux、Netty、Servlet 异步 IO 的适用边界是什么？什么时候传统阻塞模型反而更简单可靠？
