# 03 - 并发编程

---

## 一、并发模型与线程基础

1. 线程、任务、协程/虚拟线程在编程模型上有什么差异？它们分别适合什么负载？
2. 线程生命周期、阻塞、等待、超时等待对线上排查有什么意义？如何从线程栈判断线程在等什么？
3. 中断机制应该如何设计？为什么不能简单吞掉 `InterruptedException`？
4. 守护线程、优雅停机、Shutdown Hook 在线上服务中分别要注意什么？
5. CPU 密集型、IO 密集型、混合型任务的并发策略有什么不同？

## 二、共享状态与锁

1. 什么是线程安全？无状态、不可变对象、线程封闭、锁、CAS 分别如何降低并发风险？
2. `synchronized` 和 `ReentrantLock` 如何选型？可中断、超时、公平性、条件队列分别影响什么？
3. 读写锁和 `StampedLock` 适合哪些读多写少场景？乐观读为什么必须校验？
4. 死锁、活锁、饥饿分别是什么？线上如何用线程栈、锁信息和业务日志定位？
5. 锁粒度如何设计？粗锁、细锁、分段锁、无锁化分别会带来哪些取舍？
6. `volatile` 能解决什么可见性和有序性问题？为什么不能替代复合操作的原子性？

## 三、原子类、CAS 与无锁设计

1. CAS 的成功条件是什么？ABA 问题在什么场景才真正需要处理？
2. `AtomicReference`、`AtomicStampedReference`、`LongAdder`、`LongAccumulator` 分别适合什么场景？
3. 高并发计数、限流、状态机更新中如何选择锁、原子类或队列串行化？
4. 无锁算法为什么不一定比加锁更好？自旋、CPU 消耗、代码复杂度要如何权衡？
5. `Unsafe`、`VarHandle` 的定位是什么？业务代码是否应该直接使用它们？

## 四、线程池与异步编排

1. `ThreadPoolExecutor` 的核心参数如何共同决定任务执行路径？
2. 队列大小、线程数、拒绝策略如何影响接口延迟、吞吐和故障扩散？
3. 为什么不建议直接使用 `Executors` 的快捷工厂？在项目中你会如何封装线程池？
4. 线程池如何监控？你会关注活跃线程数、队列长度、拒绝次数、任务耗时还是其他指标？
5. `CompletableFuture` 如何做并行调用、依赖编排、超时、降级和异常聚合？
6. 异步任务如何传递 TraceId、用户上下文和租户上下文？`ThreadLocal` 会带来什么问题？

## 五、JUC 工具与并发容器

1. AQS 的核心思想是什么？state、队列、独占/共享模式如何支撑锁和同步器？
2. `CountDownLatch`、`CyclicBarrier`、`Semaphore`、`Phaser` 分别适合哪些协作场景？
3. `BlockingQueue`、`ConcurrentHashMap`、`CopyOnWriteArrayList` 的一致性和性能边界分别是什么？
4. `ThreadLocal` 的典型用途和内存泄漏风险是什么？在线程池、异步调用、虚拟线程下要注意什么？
5. Fork/Join 和并行流的工作窃取适合什么任务？为什么阻塞操作会破坏它的效果？

## 六、虚拟线程与现代并发

1. 虚拟线程适合解决什么问题？它对高并发阻塞 IO 服务的价值在哪里？
2. 虚拟线程不适合哪些场景？CPU 密集任务、长时间持有锁、依赖 ThreadLocal 的代码分别有什么风险？
3. 平台线程池和虚拟线程的监控、限流、资源隔离思路有什么不同？
4. Structured Concurrency 想解决什么异步编排痛点？和 `CompletableFuture` 相比有什么表达差异？
5. 使用虚拟线程改造老服务时，你会如何做压测、线程栈观察、数据库连接池和下游限流评估？

## 七、Java 内存模型（JMM）

1. JMM 试图解决什么问题？可见性、原子性、有序性在实际代码中如何体现？
2. happens-before 规则如何帮助你判断一段并发代码是否安全？
3. 指令重排序什么时候会改变程序表现？双重检查锁为什么需要 `volatile`？
4. final 字段的安全发布语义是什么？不可变对象为什么更容易并发安全？
5. 你如何向团队解释“线程安全不是靠运气测试出来的”，并用代码评审减少并发缺陷？
